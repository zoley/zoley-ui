# 实战阅读：你该怎样改这个项目

前面几篇偏“理解”，这一篇偏“动手”。你可以把它当成你的第一次项目改造指南。

## 一、如果你想新增一个页面，应该改哪里

这个项目的页面主要放在 `src/views/`。

比如你想新增一个“订单管理”页面，推荐思路是：

1. 在 `src/views` 下面建立符合语义的目录
2. 新建 `index.vue`
3. 让自动路由插件生成新路由
4. 在额外路由配置里补菜单标题、图标、排序、权限等元信息

你会发现，这个项目不是让你手写一大坨路由配置，而是更强调“按目录组织页面，再补充元信息”。

这种方式很适合后台项目，因为页面很多时，目录比长路由表更直观。

## 二、如果你想新增一个菜单，核心不是菜单组件，而是路由配置

很多新手看到左侧菜单，会下意识去改菜单组件。其实在这个项目里，菜单的上游是路由。

菜单生成逻辑在 `src/stores/modules/menu.ts`。

它会：

1. 读取 `routes`
2. 根据 `meta.hideInMenu`、`meta.roles` 过滤
3. 根据 `meta.order` 排序
4. 生成最终菜单树

所以你要新增菜单，真正要改的是：

- 页面目录
- 路由元信息
- 可能的权限码

而不是直接在组件里硬编码一个菜单项。

这就是“数据驱动 UI”的典型思路。

## 三、如果你想做权限控制，最稳妥的入口在哪里

项目里权限主要围绕 `meta.roles` 和 `authStore.roleMap`。

你可以把流程记成一句话：

“路由声明需要什么权限，登录后把用户拥有的权限装进 store，守卫和菜单都按这个判断。”

这说明权限控制主要落在三处：

1. 路由上声明规则
2. 登录后拿到权限码
3. 守卫和菜单统一消费权限码

这是行业里非常标准的做法。好处是权限规则集中，不容易失控。

## 四、如果你想接真实后端接口，第一步看哪里

先看 `src/api/request.ts`。

因为这里定义了整个项目的请求策略：

- 请求头是否带 token
- 成功时返回什么
- 失败时怎么提示
- 401 时怎么重定向

再看 `src/utils/service.ts` 和 `build/proxy/index.ts`。

这里定义了：

- 不同服务的基地址
- 开发环境代理前缀
- 真实接口地址和本地代理如何切换

这就是一个典型的前端工程思维：

- 页面不直接拼完整 URL
- 服务地址统一配置
- 开发和生产切换统一处理

## 五、如果你想改主题，别只盯着颜色

主题在这个项目里不只是“换个主色”。

`app` store 里同时管理了：

- 明暗模式
- 跟随系统
- Naive UI 主题覆盖
- 根节点 `dark` 类名
- 视图过渡动画

这说明主题系统是一个“跨组件的全局行为”，不是某个按钮自己的事。

所以你以后自己做主题功能时，也应该考虑：

1. 状态放哪里
2. 第三方组件库怎么同步
3. 自定义样式怎么同步
4. 是否要持久化
5. 是否要兼顾系统主题

## 六、如果你想让页面被缓存，应该理解 keep-alive 的前提

项目里缓存不是默认全开，而是有策略的。

你需要注意几件事：

1. 路由元信息里要允许缓存
2. 组件本身要有稳定身份
3. 页面必须仍在“有效标签上下文”中

`AppRouterView.vue` 里用的是：

- `RouterView`
- `Transition`
- `KeepAlive`

并通过 `keepAliveStore.keepAliveList` 控制 include 列表。

这是一种比“粗暴包住所有页面”更细的缓存方案。

## 七、如果你想理解这个项目的自动化能力，重点不是会用，而是知道为什么这样设计

这个项目用了几类自动化：

### 1. 自动路由

让目录结构直接变成路由。

### 2. 自动导入

减少 `ref`、`computed`、`useRoute` 之类的重复 import。

### 3. 组件自动注册

减少 UI 组件和图标组件的手工引入。

### 4. 构建时生成类型

让编辑器知道自动生成的路由名和组件类型。

这些做法的共同目标不是“炫”，而是：

- 降低重复劳动
- 降低漏配概率
- 提高一致性

你以后做项目时也可以用这个判断标准：

“一个自动化能力值不值得上，不看它酷不酷，要看它能不能稳定减少重复工作和协作错误。”

## 八、你现在最适合做的 6 个练习

### 练习 1：新增一个测试页面

目标：

- 在 `src/views/example/` 下再加一个页面
- 让它自动出现在菜单里

你会学到：

- 页面目录组织
- 自动路由生成
- 菜单如何从路由派生

### 练习 2：给页面加权限

目标：

- 给一个页面加 `meta.roles`
- 手动改一改权限数据，看菜单和路由访问如何变化

你会学到：

- 权限声明
- 守卫判断
- 菜单过滤

### 练习 3：给标签页加一个新交互

目标：

- 比如加“复制当前路径”或者“固定当前标签页”

你会学到：

- `tab` store 和 UI 组件如何协作
- 状态驱动交互的方式

### 练习 4：新增一个接口模块

目标：

- 在 `src/api` 下增加一个新接口文件
- 从页面里调用它

你会学到：

- 请求层封装如何服务页面
- 页面如何只保留最少请求细节

### 练习 5：改一个全局主题配置

目标：

- 改主色、布局宽度、过渡动画

你会学到：

- 全局配置如何流入布局和组件

### 练习 6：读一次自研插件

目标：

- 把 `packages/vite-plugin-vue-routes/` 从头到尾读一遍

你会学到：

- Vite 插件并没有想象中那么神秘
- 工程自动化本质上也是普通代码

## 九、你作为 Vue 初学者，最容易踩的几个坑

### 1. 把页面逻辑和全局逻辑混在一起

遇到“多个页面都要用”的数据或行为，先想它该不该进 store / hook / utils。

### 2. 看到自动化就害怕

自动路由、自动导入并不是黑魔法。它们本质上只是构建期帮你生成代码。

### 3. 只看模板不看数据流

页面里看起来是几个按钮、几个表格，但真正重要的是：

- 数据从哪来
- 状态存哪
- 权限谁判断
- 路由谁控制

### 4. 一上来就想全部看懂

工程项目不是一次性看懂的。你应该按模块、按问题、按场景分批次理解。

## 十、最后给你的学习建议

你现在不需要追求“把这个项目每一行都背下来”。你更应该追求的是：

1. 我知道这个项目为什么这么分层
2. 我知道新增一个功能时应该从哪层开始改
3. 我知道哪些做法是行业通用的
4. 我知道哪些是作者为了维护性做的工程抽象

当你能做到这四点时，你就已经不只是“会看 Vue 文件”，而是在真正开始理解一个前端工程项目了。

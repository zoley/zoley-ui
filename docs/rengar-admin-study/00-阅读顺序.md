# rengar-admin 阅读顺序

这份文档先帮你建立一个最重要的意识：不要一上来就钻进某个页面文件里看细节，而要先看“项目是怎么跑起来的”，再看“页面为什么会出现”，最后才看“某个按钮为什么能工作”。

## 推荐阅读路线

1. 先看项目入口
   文件：`src/main.ts`、`src/App.vue`
   目标：理解 Vue 应用是怎么创建、注册插件、挂载路由、选择布局的。

2. 再看布局系统
   文件：`src/layouts/`
   目标：理解后台系统的骨架是怎么搭起来的，比如顶部栏、侧边栏、标签页、内容区。

3. 再看路由系统
   文件：`src/router/`、`build/plugins/router.ts`、`packages/vite-plugin-vue-routes/`
   目标：理解为什么 `src/views` 下面放一个页面，最后就能变成路由。

4. 再看状态管理
   文件：`src/stores/`
   目标：理解菜单、标签页、权限、主题为什么能跨页面共享。

5. 再看接口请求层
   文件：`src/api/request.ts`、`src/utils/service.ts`、`packages/axios/`
   目标：理解 token 注入、401 处理、接口基地址切换。

6. 再看配置与构建
   文件：`vite.config.ts`、`build/plugins/`、`src/config/app.ts`
   目标：理解项目为什么有自动导入、自动路由、图标能力和主题能力。

7. 最后再看业务页面
   文件：`src/views/`
   目标：把前面学到的框架知识套进真实页面里。

## 你阅读时要一直问自己的 4 个问题

1. 这段代码属于哪一层？
   是入口层、布局层、路由层、状态层、接口层，还是页面层。

2. 它解决的是什么问题？
   不要只看“它写了什么”，更要看“没有它会出现什么麻烦”。

3. 它为什么放在这里，而不是别处？
   这能帮你建立工程分层感。

4. 这是一种项目私有写法，还是行业常见写法？
   这样你才能区分“框架基础知识”和“作者的工程设计”。

## 适合你的学习方法

如果你是 Vue 小白，我建议你按下面的节奏学：

1. 第一遍只看流程
   不要求看懂每一行，只要知道应用启动后会经过哪些步骤。

2. 第二遍按模块拆开看
   比如今天只看路由，明天只看 Pinia，后天只看布局。

3. 第三遍带着问题回看页面
   比如问自己：
   “这个菜单为什么能自动出现？”
   “这个页面为什么能被 keep-alive 缓存？”
   “这个接口为什么会自动带 token？”

4. 第四遍尝试自己改
   比如新增一个页面、一个菜单、一个权限点、一个 store。
   真正掌握是在改动中发生的。

## 这套项目最值得学的地方

- 它不是“炫技式”的大而全模板，而是偏克制、偏工程化。
- 它把“通用能力”和“业务页面”分开了，这很适合学习后台项目分层。
- 它大量使用现代 Vue 生态的常见组合：`Vue 3 + Vite + Pinia + Vue Router + Naive UI + UnoCSS`。
- 它自己做了一些轻量插件，而不是把所有逻辑堆在业务代码里，这一点很值得学习。

你后面读其他文档时，可以一直把这份“阅读地图”放在旁边参考。
